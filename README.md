# index-worldcup26-hub

## 项目简介

本仓库用于归档和发布多个独立的 HTML 页面。仓库本身不针对特定域名或具体网站，仅作为页面内容的集中存放与管理空间。所有页面均为静态 HTML 文件，可直接通过浏览器访问。

## 目录说明

```
index-worldcup26-hub/
├── pages/          # 存放所有 HTML 页面文件
├── assets/         # 页面所需的静态资源（样式、脚本、图片等）
├── docs/           # 相关文档或说明文件（可选）
└── README.md       # 本文件
```

- `pages/`：每个 HTML 文件对应一个独立页面，文件名应体现页面主题或用途。
- `assets/`：按需存放 CSS、JavaScript、图片等资源，建议按页面或功能分类组织。
- `docs/`：用于存放辅助文档、设计稿、参考链接等（非必需）。

## 页面归档说明

- 所有页面均为静态 HTML，不依赖后端服务。
- 页面间相互独立，可单独访问，不存在跨页面依赖。
- 文件命名建议使用英文小写、连字符连接，避免空格和特殊字符。
- 如需引用资源，请使用相对路径（相对于仓库根目录或 `assets/` 目录）。

## 维护说明

- 新增页面：在 `pages/` 下创建 HTML 文件，并确保相关资源正确引用。
- 更新页面：直接修改对应文件，建议保留简洁的提交记录。
- 删除页面：移除文件并确认无其他页面引用该资源。
- 请勿在页面内容中包含敏感信息、外部追踪代码或违反平台政策的元素。

## 贡献与使用

本仓库主要面向个人或小团队的页面归档需求。欢迎通过 Pull Request 提交改进，但请确保新增内容符合仓库定位和上述维护规范。
